Logic Explanation - Phase 2

Single Source of Truth: The lootArray stores all loot items as objects with name, value, and quantity. Party size is stored separately. These two pieces of state determine everything displayed on screen.

Quantity in Data Model: Added quantity property to each loot object to accurately represent multiple identical items (e.g., 3 potions). Total value is now calculated as value × quantity for each item.

Centralized Updates: updateUI() performs all calculations (totals, per-member split) and rendering (loot list, visibility, button states). It's called after every state change, ensuring the interface always reflects current data.

State-Driven Interface: Split button is enabled only when loot array has items AND party size is valid (≥1). Results are hidden when state is invalid. CSS classes control visibility instead of inline styles.

Conditional Logic: Validation prevents invalid data from entering the loot array. Party size must be valid before calculations. Empty states are handled with conditional checks and appropriate UI messages.
